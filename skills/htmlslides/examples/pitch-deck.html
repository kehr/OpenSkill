Series A pitch

Memory that scales with the work, not the calendar.

RecallOS turns ambient laptop activity into a queryable workspace memory. Built for teams that ship knowledge, not status updates.

01 / 10

Problem

Knowledge work generates artifacts faster than people can index them.

  • A typical engineer creates 200+ distinct artifacts per week (docs, comments, code, chats).
  • 87% are never read again by the author. 92% are never read by a teammate.
  • Knowledge bases capture < 4% of this -- the curated tip of the iceberg.
  • Onboarding a new hire takes ~12 weeks because the rest lives in private chats and closed tabs.
02 / 10

Why now

Three forces converged in the last 18 months.

Local-first AI, cheap vector storage on consumer machines, and a workforce that lives across 12+ SaaS tools have made workspace memory both technically feasible and economically necessary.

03 / 10

Solution

RecallOS in three layers.

Capture

Local OS-level hooks observe app windows, clipboards, and authorized API streams. Nothing leaves the machine without consent.

Index

Encrypted local vector store, indexed in 200-token chunks with cross-document linkage. Median query latency: 80ms.

Recall

Native search bar + IDE plugin + chat sidebar. One query surfaces the artifact, its context, and its history.

04 / 10

How it feels in practice

Query language is plain English (with optional filters).

$ recall "the doc about rate limiting we wrote in Q1"
[1] policy/rate-limit-design.md             March 8   you, @dana
[2] design-review notes                      March 12  Notion
[3] PR #4112 retry-with-backoff              March 15  GitHub

$ recall --since="last week" --kind=conversation \
         "what did we decide about the egress webhook?"
[1] #eng-platform 2026-05-12 14:31 (Dana, You, Sasha)
    "Lets ship it disabled by default. Flag flip next sprint."

$ recall --visualize "everything I touched related to billing in May"
